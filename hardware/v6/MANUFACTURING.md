# V6 — manufacturing package (JLCPCB)

This folder is the **fabrication-support package** for the V6 board delta. It does **not**
contain Gerbers: Gerbers/drill/CPL are generated by **EasyEDA** *after* the new parts are placed
and routed on the PCB, which is an interactive CAD step. There is currently **no V6 PCB
layout** — only the schematic delta drafted in `../README.md`. Follow the steps below to produce
a manufacturable order.

Contents:
- `BOM_v6_additions.csv` — the **new** parts only, JLCPCB BOM format (Comment, Designator, Footprint, LCSC #).
- `NETLIST_v6_additions.md` — exact net-by-net connections to add in EasyEDA.
- `CPL_v6_additions_TEMPLATE.csv` — placeholder placement list; real X/Y/rotation come from the EasyEDA export.

> The full-board V5 BOM/footprints already live in the EasyEDA project
> (`../EasyEDA_Pro/ESP32 Soil`, V5 sheet `1ca46c0a…`). V6 = V5 + the parts in `BOM_v6_additions.csv`.

## Step 1 — Schematic (EasyEDA Pro)
1. Open `../EasyEDA_Pro/ESP32 Soil`, **duplicate the V5 schematic** (sheet `1ca46c0a…`) → name it `V6`.
2. Place the parts from `BOM_v6_additions.csv` and wire them per `NETLIST_v6_additions.md`.
   Start with the **required** groups (`soil-load-switch`, `temp-humidity`); add the two
   `*-OPTIONAL` groups (`solar-sense`, `alrt-wake`) if you want them.
3. Assign the MCU pins: `IO21`=SOIL_PWR_EN, `IO16/17`=I²C (existing), `IO4`=SOLAR_SENSE,
   `IO18`=ALRT_INT. Run **ERC**; fix unconnected/conflicting nets.

## Step 2 — PCB layout
1. **Update PCB from schematic** (import changes). Keep the existing V5 stackup
   (confirm layer count in EasyEDA — likely 2-layer, 1.6 mm FR-4, 1 oz Cu).
2. Place the new parts:
   - SHT40 (U19): board **edge / near a vent, away from the LDO/charger/ESP** (self-heating → reads warm).
   - Q1 + R31/R32/C18: next to the TLC555; keep `SENSOR_3V3` short.
   - SOLAR_SENSE divider: near the `Solar+` net; keep the high-impedance node short, off the antenna keep-out.
3. Route, pour, and run **DRC against JLCPCB capabilities** (min trace/space ≥ 6/6 mil is safe,
   min drill ≥ 0.2 mm, annular ≥ 0.13 mm — JLC can do finer but 6/6 is cheap/robust).

## Step 3 — Export fabrication files (EasyEDA)
- **Fabrication → Gerber/Drill** → produces the Gerber set (RS-274X) + Excellon drill → **zip**.
- **Fabrication → BOM** and **→ Pick and Place (CPL)** → CSV each (these supersede the templates here;
  the CPL gains real Mid-X/Y/Rotation from the layout).
- Or use EasyEDA's **"One-click order"** to push straight to JLCPCB (auto-attaches Gerber/BOM/CPL).

## Step 4 — Order at JLCPCB
1. **PCB:** upload the Gerber **zip** → confirm layers/size/stackup.
2. **Assembly (PCBA, optional):** enable, upload `BOM.csv` + `CPL.csv` (the EasyEDA exports).
   - JLC matches by **LCSC part #**. The IC LCSC numbers in `BOM_v6_additions.csv` are
     known-good (AO3401A `C15127`, SHT40-AD1B `C2761795`); **verify the passive LCSC #s**
     (`C22935`/`C159966` marked "verify") against current stock — substitute same value/footprint
     if unstocked.
   - Check **rotations/polarity** in JLC's placement preview, especially **Q1 (SOT-23)** and
     **U19 (DFN-4)** — EasyEDA↔JLC rotation offsets are the usual PCBA gotcha.
3. Confirm any **"extended part"** fees (SHT40 / AO3401A may be extended vs basic).

## Reuse of legacy fab files
`../legacy_c3mini_production/` (BOM/PickAndPlace/Gerber.zip) is the **old ESP32-C3** board — a
different MCU/layout. Use it only as a JLCPCB-format reference, **not** as the V5/V6 fab source.

## Firmware pairing (already in repo)
- Soil gating: `main/sensor_data/data.c` `readMoisture()` — set `SOIL_PWR_GPIO = GPIO_NUM_21`
  (default `-1` keeps V5 unchanged), `SOIL_PWR_ACTIVE_LEVEL = 0`.
- SHT40: auto-detected at `0x44` (driver plan in `../README.md`).
- SOLAR_SENSE / ALRT: guard behind defines so a board without them is unaffected.
